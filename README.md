Fine-tuning CAMeLBERT on Arabic YouTube comments for binary sentiment classification (Negative / Positive).

## Training (notebook)

Open `notebook/arabic_sentiment_analysis.ipynb` and run all cells.

The notebook uses `CAMeL-Lab/bert-base-camelbert-mix-sentiment` with:
- [CLS] + mean-pool classification head
- Focal Loss with inverse-frequency class weights
- Layer-wise learning rate decay (LLRD)
- WeightedRandomSampler for class imbalance
- Early stopping (patience = 5)

## Batch Labeling (script)

The `script/batch_labeler.py` script labels the 138,924 unlabeled rows via the Gemini Batch API.
Each record gets a `label` (0/1) and a `confidence` score (0-1).
Use only records with `confidence >= 0.75` for training to avoid noisy labels.

### Workflow

```bash
cd script

# 1. Send 30-comment smoke test — verify everything works before the full run
python batch_labeler.py --sample

# 2. Submit all 138,924 unlabeled comments (~70 batch jobs)
python batch_labeler.py --submit

# 3. Check progress (safe to run at any time)
python batch_labeler.py --status

# 4. Once all jobs show JOB_STATE_SUCCEEDED, collect results
python batch_labeler.py --collect

# Or run the full pipeline unattended (submit + poll + collect)
python batch_labeler.py
```

State is persisted to `script/batch_state.json` after every batch submission.
**All job names are saved there** — you can always resume or inspect without losing work.

### Output format (`dataset/labeled_results.json`)

```json
[
  {
    "original_index": 1,
    "label": 1,
    "confidence": 0.92,
    "low_confidence": false
  },
  ...
]
```

### Model & cost

| Model | Why |
|---|---|
| `gemini-2.0-flash` | Best cost/quality ratio for Arabic SA. Accurate on dialectal nuance. |

`gemini-2.0-flash-lite` is ~50% cheaper but noticeably weaker on Arabic.

## Key Design Decisions

| Decision | Rationale |
|---|---|
| CAMeLBERT-Mix (not AraBERT) | Dataset contains mixed Gulf/MSA dialect YouTube comments |
| [CLS] + mean-pool head | Outperforms BiLSTM head on datasets under 50k samples |
| Confidence field in labels | Filters noisy Gemini labels before training |
| Focal Loss | Handles 60/40 class imbalance without discarding data |
| 15 comments per Gemini request | Fits easily in context window; JSON schema enforces correct output |
